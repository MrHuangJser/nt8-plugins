# ETHSessionMES 策略说明文档

> NinjaTrader 8 自动化策略 | 文件: `Strategies/ETHSessionMES.cs`

---

## 1. 策略概述

ETHSessionMES 是一个 MES（Micro E-mini S&P 500 Futures）日内交易策略。核心思路：

1. 在欧洲交易时段（ETH Session）的指定时间点以市价开仓
2. 设置基于总金额的止盈/止损
3. 若美盘开盘时仍处于浮亏状态，根据浮亏程度决定是否加仓及加仓目标（止盈或打平）
4. 美盘收盘前未触发止盈/止损则强制平仓
5. 每个交易周期只交易一次，平仓后等待下一个 ETH 时段

---

## 2. 适用品种

| 属性 | 值 |
|------|------|
| 品种 | MES (Micro E-mini S&P 500) |
| Tick Size | 0.25 点 |
| Tick Value | $1.25 / 合约 |
| Point Value | $5.00 / 合约 |

> 策略使用 `TickSize * PointValue` 动态计算，理论上适用于其他期货品种，但止盈/止损金额需相应调整。

---

## 3. 可配置参数

### 3.1 策略配置

| NinjaTrader 参数名 | 说明 | 类型 | 默认值 | 备注 |
|-------------------|------|------|--------|------|
| 交易方向 (`TradeDirection`) | 做多或做空 | 枚举 | `Long` | `Long`=买入, `Short`=卖出 |
| 入场时间 (`EntryTimeUTC8`) | UTC+8 入场时间 | HHMMSS | `80000` | 如 `80000`=08:00, `90000`=09:00 |
| 初始手数 (`InitialQty`) | 初始开仓手数 | 1-10 | `1` | ETH 时段开仓数量 |
| 加仓手数 (`AddOnQty`) | 美盘加仓手数 | 1-10 | `1` | 美盘开盘浮亏时加仓数量 |
| 加仓浮亏门槛 (`AddOnThreshold`) | 浮亏超过此金额才加仓 | $0+ | `0` | `0`=任何浮亏即加仓 |
| 打平浮亏门槛 (`BreakEvenThreshold`) | 浮亏超过此金额加仓只求打平 | $0+ | `0` | `0`=禁用，始终追求止盈 |
| 止盈金额 (`TakeProfitDollars`) | 止盈总金额 | $1+ | `150` | 所有合约合计的盈利目标 |
| 止损金额 (`StopLossDollars`) | 止损总金额 | $1+ | `2000` | 所有合约合计的最大亏损 |

### 3.2 时段配置

| NinjaTrader 参数名 | 说明 | 类型 | 默认值 | 备注 |
|-------------------|------|------|--------|------|
| 美盘开盘 (`USOpenTimeUTC8`) | 美盘开盘时间(UTC+8) | HHMMSS | `213000` | 夏令时 21:30 |
| 美盘收盘 (`USCloseTimeUTC8`) | 美盘收盘时间(UTC+8) | HHMMSS | `40000` | 夏令时次日 04:00 |
| 交易所UTC偏移 (`UtcOffsetHours`) | 交易所时区偏移 | -12~12 | `8` | 见下方时区说明 |

### 3.3 回测配置

| NinjaTrader 参数名 | 说明 | 类型 | 默认值 |
|-------------------|------|------|--------|
| 启用回测 (`EnableBacktest`) | 是否在历史数据上执行 | bool | `true` |

## 4. 时区与时间配置

### 4.1 时间参数格式

所有时间参数使用 **HHMMSS** 格式（6位整数），单位为 **UTC+8**（东八区/北京时间）：

```
080000 = 08:00:00
213000 = 21:30:00
040000 = 04:00:00
```

### 4.2 UtcOffsetHours 参数说明

此参数表示 **NinjaTrader 图表中显示的时间** 相对 UTC 的偏移量。策略内部通过此参数将 UTC+8 时间转换为图表时间。

| 图表时间 | UtcOffsetHours | 说明 |
|---------|---------------|------|
| UTC+8（北京时间） | `8` | 图表已显示北京时间，无需转换 |
| US Eastern (EDT) | `-4` | 夏令时美东时间 |
| US Eastern (EST) | `-5` | 冬令时美东时间 |
| US Central (CDT) | `-5` | 夏令时美中时间（CME 常用） |
| US Central (CST) | `-6` | 冬令时美中时间 |

### 4.3 夏/冬令时调整

| 时间节点 | 夏令时(UTC+8) | 冬令时(UTC+8) |
|---------|-------------|-------------|
| 美盘开盘 | 21:30 → `213000` | 22:30 → `223000` |
| 美盘收盘 | 次日 04:00 → `40000` | 次日 05:00 → `50000` |

> 切换夏/冬令时时需手动调整 `USOpenTimeUTC8`、`USCloseTimeUTC8` 和 `UtcOffsetHours`。

### 4.4 每日时间线（UTC+8 夏令时示例）

```
Day N
07:00 ─── ETH 可选入场窗口开始
08:00 ─── 【入场】开仓 1 手（EntryTimeUTC8=80000）
10:00 ─── ETH 可选入场窗口结束
  ...      持仓等待，TP/SL 监控中
21:30 ─── 【美盘开盘】检查浮动盈亏 → 决定是否加仓

Day N+1
  ...      持仓等待，TP/SL 监控中
04:00 ─── 【美盘收盘】未平仓则强制平仓
  ...
08:00 ─── 下一个交易周期入场
```

## 5. 交易流程

### 阶段一：ETH 开仓

在 `EntryTimeUTC8` 时刻以市价开仓 `InitialQty` 手 MES。

- **做多** (`TradeDirection = Long`): 买入开仓，信号名 `ETH_Long`
- **做空** (`TradeDirection = Short`): 卖出开仓，信号名 `ETH_Short`

开仓后立即设置止盈/止损（基于 Ticks 模式，相对成交价）。

### 阶段二：等待期（入场 → 美盘开盘）

持仓期间由 NinjaTrader Managed Approach 自动监控 TP/SL：

- **止盈触发**（浮盈 >= `TakeProfitDollars`）→ 平仓，当日交易结束
- **止损触发**（浮亏 >= `StopLossDollars`）→ 平仓，当日交易结束
- **均未触发** → 进入阶段三

### 阶段三：美盘开盘判断

在 `USOpenTimeUTC8` 时刻检查当前浮动盈亏，按以下逻辑判断：

```
                    检查浮动盈亏
                         │
          ┌──────────────┼──────────────────┐
          │              │                  │
       浮盈/持平    浮亏 < 门槛         浮亏 >= 门槛
          │              │                  │
       不加仓         不加仓              加仓
       维持单仓       维持单仓              │
                                    ┌───────┴───────┐
                                    │               │
                              浮亏<打平门槛    浮亏>=打平门槛
                                    │               │
                               正常止盈模式    打平模式
                              TP=均价+利润     TP=均价
```

**判断细节**:

1. **浮盈或持平** → 不加仓，维持原仓位和原 TP/SL
2. **浮亏但 < `AddOnThreshold`** → 不加仓（浮亏未达加仓门槛）
3. **浮亏 >= `AddOnThreshold`** → 加仓，进一步判断加仓模式：
   - 浮亏 < `BreakEvenThreshold`（或 `BreakEvenThreshold=0`）→ **正常止盈模式**，TP = 均价 + 止盈点数
   - 浮亏 >= `BreakEvenThreshold` → **打平模式**，TP = 均价（只求保本）

加仓后：
- 以两手均价为基准重新计算 TP/SL 价位
- 更新首仓和加仓仓位的 TP/SL 到相同价格
- 加仓信号名：`ETH_Long_AddOn` 或 `ETH_Short_AddOn`

### 阶段四：美盘交易期（美盘开盘 → 收盘）

- **止盈触发** → 全部平仓，当日交易结束
- **止损触发** → 全部平仓，当日交易结束
- **`USCloseTimeUTC8` 到达** → 强制市价全部平仓，当日交易结束

### 平仓后

- 计算下一个 `EntryTimeUTC8` 时刻作为下次入场时间
- 重置所有状态标志
- 若有打平模式交易完成，打平计数器 +1

## 6. 止盈止损计算

### 6.1 单仓阶段

以默认参数（`InitialQty=1`, `TP=$150`, `SL=$2000`）为例：

```
每 tick 价值 = TickSize × PointValue = 0.25 × $5 = $1.25

TP ticks = $150 / (1 × $1.25) = 120 ticks = 30 点
SL ticks = $2000 / (1 × $1.25) = 1600 ticks = 400 点
```

| 方向 | 止盈价 | 止损价 |
|------|--------|--------|
| LONG | 成交价 + 30 点 | 成交价 - 400 点 |
| SHORT | 成交价 - 30 点 | 成交价 + 400 点 |

### 6.2 双仓阶段（加仓后 - 正常止盈模式）

以默认参数（`InitialQty=1`, `AddOnQty=1`, `TP=$150`, `SL=$2000`）为例：

```
均价 = (首仓成交价 + 加仓成交价) / 2
总手数 = 2

TP 点数 = $150 / (2 × $5) = 15 点
SL 点数 = $2000 / (2 × $5) = 200 点
```

| 方向 | 止盈价 | 止损价 |
|------|--------|--------|
| LONG | 均价 + 15 点 | 均价 - 200 点 |
| SHORT | 均价 - 15 点 | 均价 + 200 点 |

### 6.3 双仓阶段（加仓后 - 打平模式）

打平模式下 TP 点数为 0：

| 方向 | 止盈价 | 止损价 |
|------|--------|--------|
| LONG | 均价 | 均价 - 200 点 |
| SHORT | 均价 | 均价 + 200 点 |

> **注意**: 打平模式的 TP = 均价意味着两手仓位的总盈亏为 $0（不考虑手续费和滑点）。实际上首仓会有亏损，加仓会有等额盈利。

## 7. 场景示例

以 `EntryTimeUTC8=80000`, `TradeDirection=Long`, `TP=$150`, `SL=$2000`, `AddOnThreshold=$50`, `BreakEvenThreshold=$200` 为例。

### 场景 A：ETH 时段止盈

| 时间(UTC+8) | 事件 | 价格 | 盈亏 |
|-------------|------|------|------|
| 08:00 | 买入 1 手 MES | 5,500.00 | - |
| 08:00 | TP=5,530.00, SL=5,100.00 | - | - |
| 14:35 | 触及 5,530.00，止盈平仓 | 5,530.00 | **+$150** |

**结果**: 当日结束，等待次日 08:00。

### 场景 B：美盘开盘浮亏未达门槛，不加仓

| 时间(UTC+8) | 事件 | 价格 | 盈亏 |
|-------------|------|------|------|
| 08:00 | 买入 1 手 MES | 5,500.00 | - |
| 21:30 | 美盘开盘，浮亏 $30 | 5,494.00 | -$30 |
| 21:30 | 浮亏 $30 < 门槛 $50 → 不加仓 | - | - |
| 23:00 | 触及 5,530.00，止盈平仓 | 5,530.00 | **+$150** |

**结果**: 浮亏未达加仓门槛，维持单仓止盈。

### 场景 C：加仓 + 正常止盈模式

| 时间(UTC+8) | 事件 | 价格 | 盈亏 |
|-------------|------|------|------|
| 08:00 | 买入 1 手 MES | 5,500.00 | - |
| 21:30 | 美盘开盘，浮亏 $100 | 5,480.00 | -$100 |
| 21:30 | 浮亏 $100 >= 门槛 $50，且 < 打平门槛 $200 | - | - |
| 21:30 | 加仓 1 手 @ 5,480.00 | - | - |
| 21:30 | 均价=5,490.00, TP=5,505.00, SL=5,290.00 | - | - |
| 23:15 | 触及 5,505.00，止盈平仓 2 手 | 5,505.00 | **+$150** |

**结果**: 加仓后正常止盈。

### 场景 D：加仓 + 打平模式

| 时间(UTC+8) | 事件 | 价格 | 盈亏 |
|-------------|------|------|------|
| 08:00 | 买入 1 手 MES | 5,500.00 | - |
| 21:30 | 美盘开盘，浮亏 $300 | 5,440.00 | -$300 |
| 21:30 | 浮亏 $300 >= 打平门槛 $200 → 打平模式 | - | - |
| 21:30 | 加仓 1 手 @ 5,440.00 | - | - |
| 21:30 | 均价=5,470.00, **TP=5,470.00**(均价), SL=5,270.00 | - | - |
| 01:20 | 触及 5,470.00，打平平仓 2 手 | 5,470.00 | **$0** |

**结果**: 打平出场，打平计数器 +1。

### 场景 E：加仓后止损

| 时间(UTC+8) | 事件 | 价格 | 盈亏 |
|-------------|------|------|------|
| 08:00 | 买入 1 手 MES | 5,500.00 | - |
| 21:30 | 美盘开盘，浮亏 $150 | 5,470.00 | -$150 |
| 21:30 | 加仓 1 手 @ 5,470.00，均价=5,485.00 | - | - |
| 21:30 | TP=5,500.00, SL=5,285.00 | - | - |
| 01:40 | 触及 5,285.00，止损平仓 2 手 | 5,285.00 | **-$2,000** |

**结果**: 止损出场。

### 场景 F：收盘强制平仓

| 时间(UTC+8) | 事件 | 价格 | 盈亏 |
|-------------|------|------|------|
| 08:00 | 买入 1 手 MES | 5,500.00 | - |
| 21:30 | 美盘开盘，浮盈 $50 → 不加仓 | 5,510.00 | +$50 |
| 04:00 | 美盘收盘，强制平仓 | 5,518.00 | **+$90** |

**结果**: 未达止盈但有盈利，收盘平仓。

## 10. 技术实现细节

### 10.1 NinjaTrader 框架配置

| 属性 | 值 | 说明 |
|------|------|------|
| `Calculate` | `OnPriceChange` | 每次价格变化触发（但逻辑仅在 `IsFirstTickOfBar` 时执行） |
| `EntriesPerDirection` | `2` | 允许同方向 2 笔入场（首仓 + 加仓） |
| `EntryHandling` | `UniqueEntries` | 每个信号名独立管理 |
| `IsExitOnSessionCloseStrategy` | `false` | 不使用 NT 内置的收盘平仓 |
| `RealtimeErrorHandling` | `IgnoreAllErrors` | 忽略实时错误避免策略终止 |

### 10.2 订单管理

| 订单信号名 | 用途 |
|-----------|------|
| `ETH_Long` / `ETH_Short` | 首仓入场 |
| `ETH_Long_AddOn` / `ETH_Short_AddOn` | 加仓入场 |
| `ForceClose` | 美盘收盘强制平仓 |

### 10.3 TP/SL 模式切换

- **单仓阶段**: 使用 `CalculationMode.Ticks`（相对成交价的 ticks 数）
- **加仓后**: 切换为 `CalculationMode.Price`（绝对价格），因为需要两手仓位共用同一个 TP/SL 价格

加仓成交后会在 `OnExecutionUpdate` 中用精确成交价重新计算 TP/SL 价位，覆盖 `CheckAddOn` 中基于估算价格的初步设置。

### 10.4 时间转换

`ToNextExchangeTime(afterExchange, utc8HHMMSS)` 方法：
1. 将交易所本地时间转为 UTC+8
2. 在 UTC+8 时间轴上构建目标时刻
3. 若目标已过则推到下一天
4. 转回交易所本地时间

自动处理跨午夜场景（如 UTC+8 的 04:00 在交易所时间可能是前一天的 15:00）。

## 11. 风险分析

### 11.1 单日最大风险

| 指标 | 值（默认参数） |
|------|--------------|
| 最大单日亏损 | $2,000 |
| 最大持仓 | 2 手 MES |
| 最大持仓时间 | ~20 小时（08:00 → 次日 04:00） |
| 最大保证金占用 | ~$2,640（2 手 MES 日内保证金，视券商而定） |

### 11.2 风险收益比

| 场景 | 最大盈利 | 最大亏损 | 盈亏比 | 盈亏平衡胜率 |
|------|---------|---------|--------|-------------|
| 单仓止盈 | $150 | $2,000 | 1:13.3 | ~93% |
| 双仓止盈 | $150 | $2,000 | 1:13.3 | ~93% |
| 打平模式 | $0 | $2,000 | - | 不贡献盈利 |

> 实际胜率需求低于 93%，因为收盘平仓（场景 F）可能产生不等额的盈亏，打平模式也会避免部分亏损转化为止损。

### 11.3 风险要点

1. **持仓时间长**: 从 ETH 到美盘收盘跨度大，期间可能遇到重大新闻/数据
2. **滑点风险**: 止损 $2,000 在极端行情（如跳空）下可能因滑点扩大
3. **加仓后波动空间收窄**: 止损从 400 点收窄至 200 点（从均价算）
4. **ETH 流动性**: ETH 时段流动性低于美盘，入场可能有较大点差
5. **打平模式不贡献盈利**: 打平模式仅避免亏损扩大，但不产生利润

## 12. 参数优化方向

| 参数 | 优化方向 | 说明 |
|------|---------|------|
| `EntryTimeUTC8` | 07:00 / 08:00 / 09:00 / 10:00 | 对比不同入场时间的胜率和收益 |
| `TradeDirection` | 动态化 | 可根据技术指标或前日趋势选择方向 |
| `TakeProfitDollars` | $100 ~ $500 | 调整盈亏比，更大 TP 降低胜率需求 |
| `StopLossDollars` | $500 ~ $2000 | 更小 SL 提高盈亏比但降低胜率 |
| `AddOnThreshold` | $0 ~ $500 | 过滤微小浮亏的无效加仓 |
| `BreakEvenThreshold` | $0 ~ $500 | 区分轻度和重度浮亏的加仓策略 |

## 13. 使用步骤

1. 将 `ETHSessionMES.cs` 放入 NinjaTrader 的 `Strategies` 目录
2. 在 NinjaTrader 中编译（F5 或菜单 NinjaScript → Compile）
3. 在 MES 图表上右键 → Strategies → 添加 `ETHSessionMES`
4. 配置参数：
   - 设置 `TradeDirection`（做多/做空）
   - 设置 `EntryTimeUTC8`（入场时间）
   - 根据图表时区设置 `UtcOffsetHours`
   - 调整夏/冬令时的 `USOpenTimeUTC8` 和 `USCloseTimeUTC8`
5. 回测模式下确保 `EnableBacktest = true`
6. 实盘前建议先在模拟账户中运行验证

> **推荐 K 线周期**: 1 分钟或 5 分钟，确保时间触发点精度足够。
